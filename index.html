Workforce Attendance Dashboard Enhancement
TP-Link Network Monitoring + API vs. Production Data Comparison Module
Two major capabilities have been added to the internal workforce attendance dashboard in this release, significantly enhancing its utility as a daily operational tool:

TP-Link Network Monitoring Integration — real-time device and network connectivity status is now visible directly within the dashboard. On-floor managers no longer need to switch between tools; network health across all attendance terminals and connected devices is monitored from a single pane of glass, with instant alerts if a device goes offline.

API vs. Production Data Comparison Module — a dedicated module cross-references attendance records fetched from the Presence360 API against the production-level data maintained by the HR team. This surfaces discrepancies instantly, highlights records that are present in one source but missing in another, and provides a clearer and more reliable view of ground-level attendance. The dashboard is now moving closer to being the single source of truth for daily workforce visibility.

SMT Feeder Automation Upgrade
Automatic Alternate Part Code Mapping from BOM Files
The feeder list automation tool — currently in active daily use by the SMT team — is being upgraded with a critical new capability.

Currently, when a BOM contains alternate part codes, engineers must manually look up and substitute the correct feeder assignment before setup. This upgrade eliminates that step entirely: the tool will now intelligently detect and extract alternate part codes directly from BOM files during feeder list generation, resolving them automatically without engineer intervention.

This reduces the risk of feeder setup errors caused by part code mismatches, shortens line changeover preparation time, and removes a category of manual errors that previously required rework. The enhancement is currently under active development and is targeted for release in the next sprint, after which it will be rolled out to the SMT team for validation.
In Development

Presence360 Attendance Issue Escalation
Punch Record Sync Failure — Formally Escalated to Support
A critical data synchronisation failure has been identified on the Presence360 platform: employee punch records are not reaching the portal or the API, despite physical punches being successfully recorded at attendance devices on the factory floor — as confirmed by the HR team through manual log verification.

This behaviour points to a device-to-server synchronisation failure on Presence360's infrastructure, not at the device level. The physical devices are functioning correctly and recording punches locally. The failure occurs in the transmission layer between device and server.

Formal communication has been initiated with the Presence360 support team with full supporting documentation, including log evidence, affected device IDs, timestamps, and a detailed description of the issue. The matter has been officially escalated and is pending resolution from their side. The team will be kept informed as developments unfold and a workaround is being assessed internally in the interim.
Escalated
